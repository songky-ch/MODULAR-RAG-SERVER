# Modular RAG Server 技术学习课程

## 课程定位

本课程以 `MODULAR-RAG-SERVER` 为真实工程案例, 目标不是学会运行、修改或复刻该项目, 而是掌握其背后可迁移的技术思想、原理与工程实践。

学习完成后应能够:

- 理解文档如何被加工成可检索的数据资产;
- 掌握 Dense、BM25、混合检索、RRF 和 Rerank 的原理;
- 理解可插拔架构、幂等处理、优雅降级与可观测性;
- 用 Golden Test Set 和量化指标评估检索系统;
- 阅读同类 AI 与搜索工程源码;
- 将这些技术迁移到自己的项目。

## 项目能力边界

当前仓库的查询主链路主要完成检索、融合、重排和引用封装。`ResponseBuilder` 返回的主要是检索片段, 不是完整的生成式回答。因此本课程将其定位为“RAG 检索基础设施案例”, 不把现有实现夸大为完整的生产级 RAG 闭环。

## 14 天课程安排

### Day 1: 建立 RAG 系统的正确心智模型

- 学习目标: 区分文档处理、检索、重排、生成和引用, 理解完整 RAG 链路及本仓库的能力边界。
- 理论: RAG 价值、参数知识与外部知识、Index-time 与 Query-time、召回错误与生成错误。
- 代码案例: `src/core/types.py`、`src/ingestion/pipeline.py`、`src/mcp_server/tools/query_knowledge_hub.py`。
- 实践: 画出最小 RAG 系统, 标注离线/在线阶段、输入输出和错误类型。
- 成果: 能准确解释 RAG 各阶段与当前项目的真实边界。

### Day 2: 数据契约与流水线设计

- 学习目标: 理解长流水线为什么必须有稳定中间表示。
- 理论: Pipeline、数据契约、中间表示、阶段职责与数据血缘。
- 代码案例: `Document`、`Chunk`、`ChunkRecord` 和 `IngestionPipeline.run`。
- 实践: 实现 `RawText → CleanDocument → Paragraph[] → EnrichedParagraph[]` 小型流水线。
- 成果: 能为多阶段数据系统设计稳定契约。

### Day 3: 文档分块与上下文工程

- 学习目标: 掌握 Chunk size、overlap 和语义边界的权衡。
- 理论: 固定长度、递归分块、结构边界、overlap、Parent-Child Chunking。
- 代码案例: `DocumentChunker`、`RecursiveSplitter`、`FinancialReportSplitter`。
- 实践: 比较三组块大小与 overlap 策略对信息完整性的影响。
- 成果: 能根据文档结构选择分块策略。

### Day 4: Embedding 与向量空间

- 学习目标: 理解 Embedding、相似度和向量检索。
- 理论: 文本向量化、向量维度、余弦相似度、归一化与索引重建。
- 代码案例: `BaseEmbedding`、`EmbeddingFactory`、`DenseEncoder`、`DenseRetriever`。
- 实践: 编码10句话并计算相似度矩阵。
- 成果: 能解释向量检索为何有效及它的失败边界。

### Day 5: BM25 与稀疏检索

- 学习目标: 掌握倒排索引、TF-IDF 与 BM25。
- 理论: Tokenization、TF、DF、IDF、长度归一化、BM25 参数与中文 n-gram。
- 代码案例: `SparseEncoder`、`BM25Indexer`、`QueryProcessor._tokenize`、`SparseRetriever`。
- 实践: 为5篇短文实现简化 BM25。
- 成果: 能手算 BM25 得分并判断它何时优于向量检索。

### Day 6: 混合检索与多路召回

- 学习目标: 理解生产搜索系统为什么通常采用多路召回。
- 理论: 召回率与精确率、Candidate Generation、并行扇出、Top-K 预算与延迟。
- 代码案例: `HybridSearch.search`、`_run_retrievals`、`_run_parallel_retrievals`。
- 实践: 比较语义查询、精确编号查询和混合查询的三种检索结果。
- 成果: 能为不同检索路径分配候选预算。

### Day 7: RRF 排序融合

- 学习目标: 理解不同分数量纲下的排序融合。
- 理论: 分数融合、排名融合、RRF 公式、参数 $k$ 和确定性排序。
- 代码案例: `RRFFusion.fuse` 及 `test_fusion_rrf.py`。
- 实践: 手工实现 RRF 并融合两组排名列表。
- 成果: 能独立实现 RRF 并解释它为什么不依赖原始分数量纲。

### Day 8: 两阶段排序与 Rerank

- 学习目标: 理解召回与精排拆分的必要性。
- 理论: Bi-Encoder、Cross-Encoder、LLM-as-Reranker、成本与精度、Top-K 与 Top-N。
- 代码案例: `CoreReranker`、`CrossEncoderReranker`、`LLMReranker`。
- 实践: 用快速方法召回10个候选, 再用联合评分排出前3名。
- 成果: 能设计合理的召回—精排两阶段系统。

### Day 9: 可插拔架构与依赖倒置

- 学习目标: 掌握如何隔离易变的基础设施。
- 理论: ABC、Strategy、Factory、Adapter、依赖注入和配置边界。
- 代码案例: Embedding、Splitter、VectorStore 和 Reranker 的 Base 类与 Factory。
- 实践: 实现可切换的内存搜索和文件搜索。
- 成果: 能识别变化点并用最小抽象隔离它。

### Day 10: 幂等、确定性 ID 与增量索引

- 学习目标: 掌握可重复执行的数据处理设计。
- 理论: 幂等、内容哈希、业务主键、Upsert、成功标记时机和部分写入风险。
- 代码案例: `SQLiteIntegrityChecker`、`DocumentChunker`、`VectorUpserter`、`mark_success`。
- 实践: 实现可重复运行、内容变化后重新处理的文本索引器。
- 成果: 能设计可安全重试的数据任务。

### Day 11: 故障隔离与优雅降级

- 学习目标: 理解可用性设计与“捕获所有异常”的区别。
- 理论: 核心/增强能力、Fail-fast、Fail-soft、部分结果、超时和降级状态。
- 代码案例: `HybridSearch`、`CoreReranker`、`CompositeEvaluator` 的回退逻辑。
- 实践: 实现两个并行数据源, 仅在两路都失败时报错。
- 成果: 能设计有明确语义的降级策略。

### Day 12: Trace 可观测性与 RAG 诊断

- 学习目标: 能从结果错误反推出问题阶段。
- 理论: Logs、Metrics、Traces、Stage Span、Correlation ID、输入输出快照和延迟分解。
- 代码案例: 查询链的 `query_processing → dense/sparse → fusion → rerank` 和摄取链 Trace。
- 实践: 给 Day 11 练习增加阶段 Trace。
- 成果: 能判断问题属于解析、召回、融合、重排还是服务层。

### Day 13: 离线评估与质量回归

- 学习目标: 掌握如何用数据判断 RAG 是否变好。
- 理论: Golden Test Set、Hit Rate@K、Recall@K、MRR、检索/生成评估分层和回归测试。
- 代码案例: `GoldenTestCase`、`EvalRunner`、`CustomEvaluator`、`CompositeEvaluator`、`RagasEvaluator`。
- 实践: 建立10条 Golden Query, 比较两组参数或 Token 策略。
- 成果: 能建立最小但可信的检索评估闭环。

### Day 14: MCP 工具化与系统综合设计

- 学习目标: 理解如何把内部能力转换为 AI 可发现、可调用、可验证的工具。
- 理论: Tool Schema、参数约束、协议/业务分层、结构化输出、引用、异步边界和阻塞 I/O 隔离。
- 代码案例: `ProtocolHandler`、`create_mcp_server`、`QueryKnowledgeHubTool`、`ResponseBuilder`、`CitationGenerator`。
- 实践: 把 Day 13 的检索器封装成带 JSON Schema、来源、分数和降级信息的独立工具。
- 成果: 能把内部技术能力封装成适合 Agent 调用的稳定接口。

## 教学执行规则

1. 严格按 Day 1 至 Day 14 顺序教学。
2. 每天以技术知识点为主线, 不按项目目录或运行流程教学。
3. 每日内容必须包含技术背景、核心原理、项目案例、通用工程经验和脱离项目的实践练习。
4. 每天正式开课时, 在当前目录新建一个独立 Markdown 文档。
5. 每日文档命名格式: `01-Day1-RAG系统心智模型.md` 至 `14-Day14-MCP工具化与系统设计.md`。
6. 每日文档记录当天理论、源码案例、实践任务、作答/反馈、掌握情况与复习结论。
7. 如课程计划发生调整, 立即同步更新本文档及已建立的受影响每日文档。
8. 未开始的课程不预先创建空文档。
9. 后续课程产生的 Markdown、练习代码、测试代码和其他学习产物, 统一按用户的个人代码仓策略管理, 不提交到案例项目原有 Git 远端。
10. 首次需要上传时, 在用户的个人账号下新建公开仓库, 确认远端归属后再提交课程文档与代码。

## 学习验收标准

14 天后应能够独立完成以下任务:

- 解释复杂文档如何转换为稳定、可追溯的检索数据;
- 根据文档与查询特征设计分块和检索策略;
- 解释 Dense、BM25、RRF 和 Rerank 的职责边界;
- 设计可替换的基础设施抽象;
- 设计可重试、可降级、可追踪的处理链;
- 用 Golden Test Set 评估修改是提升还是退化;
- 把内部技术能力封装为稳定的 AI 工具接口。
